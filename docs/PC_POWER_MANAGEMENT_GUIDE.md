# 💻 PC 전원 관리 및 자동매매 시스템 운영 가이드

## 📋 개요

이 문서는 크롬 익스텐션 기반 자동매매 시스템이 다양한 PC 전원 상태에서 어떻게 작동하는지 설명하고, 24시간 안정적인 운영을 위한 최적 설정 방법을 제공합니다.

---

## 🔋 PC 전원 상태별 동작 분석

### ✅ 정상 작동 상태

#### 1. 일반 사용 상태
- **상태**: PC 켜짐 + 모니터 켜짐 + 브라우저 활성
- **익스텐션 동작**: ✅ **완전 정상 작동**
- **폴링 간격**: 3초 (설정값 그대로)
- **매크로 실행**: ✅ 즉시 실행
- **특이사항**: 없음

#### 2. 화면보호기 상태
- **상태**: PC 켜짐 + 화면보호기 실행 + 브라우저 실행 중
- **익스텐션 동작**: ✅ **완전 정상 작동**
- **폴링 간격**: 3초 (변화 없음)
- **매크로 실행**: ✅ 정상 실행 (화면 깨우지 않음)
- **특이사항**: 
  - 매크로 실행 시 화면이 깨어나지 않음
  - 백그라운드에서 조용히 거래 실행
  - 24시간 운영에 최적

#### 3. 모니터 꺼짐 상태
- **상태**: PC 켜짐 + 모니터 전원 꺼짐 + 브라우저 실행 중
- **익스텐션 동작**: ✅ **완전 정상 작동**
- **폴링 간격**: 3초 (변화 없음)
- **매크로 실행**: ✅ 정상 실행 (모니터 켜지지 않음)
- **특이사항**:
  - 전력 소모 최소화
  - 모니터 수명 연장
  - 소음 없는 24시간 운영

### ⚠️ 제한적 작동 상태

#### 4. 브라우저 최소화 상태
- **상태**: PC 켜짐 + 브라우저 최소화 (작업표시줄에만 표시)
- **익스텐션 동작**: ⚠️ **성능 저하 가능**
- **폴링 간격**: 3초 → 10초 이상으로 증가 가능
- **매크로 실행**: ⚠️ 지연 발생 가능
- **특이사항**:
  - Chrome의 백그라운드 탭 성능 제한 정책 적용
  - 중요한 거래 기회 놓칠 수 있음
  - **권장하지 않음**

#### 5. 브라우저 숨김 상태 (다른 창이 전체화면)
- **상태**: PC 켜짐 + 브라우저 실행 중이지만 다른 앱이 전체화면
- **익스텐션 동작**: ⚠️ **성능 저하 가능**
- **폴링 간격**: 3초 → 5-10초로 증가 가능
- **매크로 실행**: ⚠️ 약간의 지연 가능
- **특이사항**:
  - 게임, 동영상 등 전체화면 앱 실행 시
  - 일시적 성능 저하 발생

### ❌ 완전 중단 상태

#### 6. 절전 모드 (Sleep)
- **상태**: PC 절전 모드 진입
- **익스텐션 동작**: ❌ **완전 중단**
- **폴링 간격**: 중단됨
- **매크로 실행**: ❌ 불가능
- **특이사항**:
  - RAM 내용은 유지되지만 CPU 중단
  - 네트워크 연결 중단
  - 자동매매 완전 불가능

#### 7. 최대절전 모드 (Hibernate)
- **상태**: PC 최대절전 모드 진입
- **익스텐션 동작**: ❌ **완전 중단**
- **폴링 간격**: 중단됨
- **매크로 실행**: ❌ 불가능
- **특이사항**:
  - RAM 내용을 디스크에 저장 후 완전 종료
  - 전원 완전 차단
  - 자동매매 완전 불가능

#### 8. PC 종료
- **상태**: PC 완전 종료
- **익스텐션 동작**: ❌ **완전 중단**
- **폴링 간격**: 중단됨
- **매크로 실행**: ❌ 불가능
- **특이사항**: 당연히 모든 작업 중단

## ⚙️ 24시간 자동매매를 위한 최적 설정

### 🎯 권장 설정 (최적화)

#### Windows 전원 설정:
```
제어판 → 전원 옵션 → 계획 설정 변경

✅ 권장 설정:
- 디스플레이 끄기: 30분 (또는 원하는 시간)
- 컴퓨터를 절전 모드로: 안함

고급 전원 설정:
- USB 선택적 일시 중단: 사용 안 함
- 하드 디스크 끄기: 안함
- 무선 어댑터 설정: 최고 성능
- 절전 모드: 안함
- 최대절전 모드: 사용 안 함
```

#### Chrome 브라우저 설정:
```
✅ 권장 설정:
- 거래소 탭을 고정 탭으로 설정
- 다른 불필요한 탭들 정리 (메모리 절약)
- Chrome을 시작 프로그램에 등록
- "Chrome이 백그라운드에서 계속 실행" 옵션 활성화
```

#### 시스템 설정:
```
✅ 권장 설정:
- 자동 업데이트 시간: 새벽 4-5시로 설정
- 바이러스 검사 시간: 새벽 3-4시로 설정
- 디스크 조각 모음: 주말 새벽으로 설정
- 시스템 복원 지점: 주간 단위로 설정
```

### 🚫 피해야 할 설정

#### 절대 하지 말아야 할 설정:
```
❌ 금지 설정:
- 절전 모드 활성화
- 최대절전 모드 활성화
- USB 절전 모드 활성화
- 하드디스크 자동 끄기 활성화
- Chrome 백그라운드 실행 비활성화
```

#### 주의해야 할 설정:
```
⚠️ 주의 설정:
- 브라우저 최소화 (성능 저하 가능)
- 전체화면 앱 장시간 실행
- 메모리 부족 상태 방치
- 네트워크 불안정 상태 방치
```

## 🔧 실제 운영 시나리오

### 💡 시나리오 1: 일반 가정용 PC (권장)

#### 설정:
- **낮 시간**: 일반 사용 + 자동매매 병행
- **밤 시간**: 화면보호기 + 자동매매 지속
- **모니터**: 30분 후 자동 꺼짐
- **PC**: 24시간 켜둠 (절전 모드 비활성화)

#### 장점:
- ✅ 24시간 안정적 자동매매
- ✅ 전력 소모 최적화
- ✅ 하드웨어 수명 연장
- ✅ 소음 최소화

#### 단점:
- ⚠️ 전기료 증가 (월 2-3만원)
- ⚠️ 하드웨어 24시간 가동

### 💡 시나리오 2: 전용 트레이딩 PC (최적)

#### 설정:
- **용도**: 자동매매 전용
- **모니터**: 항상 꺼짐 또는 연결 안함
- **브라우저**: 거래소 탭만 열어둠
- **기타 프로그램**: 최소한만 설치

#### 장점:
- ✅ 최고 성능 및 안정성
- ✅ 간섭 요소 최소화
- ✅ 전력 소모 최적화
- ✅ 보안성 향상

#### 단점:
- ⚠️ 별도 PC 필요 (초기 비용)
- ⚠️ 관리 포인트 증가

### 💡 시나리오 3: VPS 서버 (고급)

#### 설정:
- **환경**: 클라우드 VPS (Windows)
- **접속**: 원격 데스크톱
- **관리**: 스마트폰으로 모니터링

#### 장점:
- ✅ 24시간 안정적 운영
- ✅ 정전/인터넷 장애 대비
- ✅ 전기료 절약
- ✅ 원격 관리 가능

#### 단점:
- ⚠️ 월 사용료 (3-10만원)
- ⚠️ 설정 복잡도 증가
- ⚠️ 네트워크 지연 가능

### 💡 전기료 계산

#### 일반 데스크톱 PC:
```
소비전력: 약 200W (모니터 제외)
24시간 × 30일 = 720시간
720시간 × 0.2kW × 전기료(약 130원/kWh) = 약 18,720원/월
```

#### 저전력 미니 PC:
```
소비전력: 약 50W
24시간 × 30일 = 720시간  
720시간 × 0.05kW × 130원/kWh = 약 4,680원/월
```

#### 모니터 끄기 효과:
```
모니터 소비전력: 약 30-50W
월 절약 금액: 약 2,800-4,680원
```

## 🎯 최종 권장사항

### ✅ 24시간 자동매매 최적 설정

#### 시스템 설정:
1. **절전 모드**: 완전 비활성화
2. **화면보호기**: 30분 후 활성화 (선택사항)
3. **모니터**: 30분 후 자동 꺼짐
4. **브라우저**: 항상 실행 상태 유지
5. **거래소 탭**: 고정 탭으로 설정

#### 운영 방식:
1. **주간**: 일반 사용 + 자동매매 병행
2. **야간**: 화면보호기 + 자동매매 지속
3. **주말**: 24시간 자동매매 운영
4. **점검**: 주 1회 시스템 재부팅

#### 안전 장치:
1. **UPS**: 정전 대비 (권장)
2. **이중 인터넷**: 유선 + 무선 백업
3. **모바일 앱**: 비상시 수동 거래
4. **알림 설정**: 중요 상황 즉시 알림

### 🏆 성공적인 24시간 자동매매 운영의 핵심

1. **안정성 > 성능**: 화려한 기능보다 안정적 동작이 우선
2. **모니터링**: 정기적인 시스템 점검과 성과 분석
3. **백업 계획**: 비상 상황 대비 계획 수립
4. **점진적 개선**: 작은 문제들을 지속적으로 개선
5. **리스크 관리**: 과도한 레버리지나 포지션 크기 지양
